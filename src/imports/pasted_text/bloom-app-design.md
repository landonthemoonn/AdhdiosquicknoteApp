Design a mobile-first iOS app interface for a product called “Bloom” — a real-time cognitive assistant designed for fast-paced environments and ADHD users.

The core concept is: capture messy input (especially voice) and instantly convert it into clear, structured actions.

STYLE DIRECTION:
- Glassmorphism UI with soft blur backgrounds
- Gradient mesh backgrounds (deep red, peach, warm gray, subtle neon accents)
- Rounded corners (16–24px radius)
- Minimal, high-contrast typography (Inter or SF Pro)
- Clean, modern, slightly futuristic but grounded
- Avoid overly playful or childish UI — this should feel premium and intelligent
- Use subtle glow effects and soft shadows for depth
- Overall aesthetic: “calm control in chaos”

PRIMARY SCREEN: AUDIO CAPTURE MODE

1. CENTER CORE (main interaction)
- Large circular “Bloom Core” button in center
- Frosted glass style with iridescent gradient (cyan → peach → pink → lavender)
- Soft glow
- Label: “Hold to record” or tap to start recording
- On active state: slightly scales down and pulses

2. LIVE RECORDING STATE
- Top: waveform visualization (clean, minimal)
- Below: live transcription text streaming in real time (not perfect, slightly faded older text)
- Bottom: quick tag chips:
  - NOW
  - NEXT
  - LATER
  - NOTE
  - QUESTION
- Floating “Mark moment” button

3. POST-RECORDING SCREEN (MOST IMPORTANT)

Immediately after recording ends, show a clean action-based layout:

Four sections displayed as soft glass cards:

🔴 NOW
- bold, high priority tasks

🟡 NEXT
- secondary actions

⚪ LATER
- deferred or follow-up items

🧠 NOTE
- context or observations

Each item should look like a tappable card with:
- short text
- subtle icon
- slight elevation

IMPORTANT:
- No long paragraphs
- No full transcript shown initially
- Focus on clarity and scannability

4. EXPANDED VIEW (tap into a capture)

- Show:
  - Summary (short paragraph)
  - Full transcript (collapsible)
  - Extracted actions
  - Pattern detection (e.g. “recurring issue detected”)
- Include Bloom action buttons:
  - NOW
  - NOT NOW
  - MAKE SMALLER
  - DRAFT
  - REFERENCE

5. NAVIGATION

- Bottom minimal nav:
  - Capture (center)
  - Tasks
  - Playbook
- Floating, lightweight, non-intrusive

INTERACTION DETAILS:
- Smooth transitions between states
- Micro animations on tap (scale, fade, glow)
- Maintain a feeling of speed and responsiveness
- Design for one-handed use

TONE:
- Calm, intelligent, empowering
- Designed for someone under pressure who needs instant clarity

DO NOT:
- Overcrowd UI
- Show complex tables or heavy data views
- Rely on dense text
- Introduce unnecessary features

GOAL:
Make the user feel like their brain just got quieter and clearer within seconds.
